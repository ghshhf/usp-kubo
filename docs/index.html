核心特性

为什么选择 USP？

专为现代应用设计的多后端存储解决方案

🔄

多后端支持

同时支持 Local、P2P (Kademlia DHT)、S3 兼容存储和 IPFS。 根据数据特征自动选择最优存储位置。

🎯

策略路由

基于键名模式、文件大小、标签自动路由。 小文件走本地，大文件上云，热数据用 P2P 缓存。

🔌

离线降级

去中心化存储支持离线模式。IPFS 不可用时自动切换到本地缓存， 恢复连接后同步到网络。

🛡️

S3 完整兼容

手写 AWS SigV4 签名，支持所有 S3 兼容服务。 可对接 MinIO、Cloudflare R2、Ceph 等。

⚡

Tokio 异步

纯异步架构，基于 Tokio 运行时。 高性能、低延迟，充分利用现代硬件。

📝

灵活配置

支持 .usp.toml 配置文件、环境变量覆盖、 运行时动态调整。

存储后端

四大存储引擎

每种后端针对不同场景优化，按需组合

💾

LocalBackend

Local

本地文件系统存储，使用 libc::statvfs 获取真实磁盘空间。 支持目录递归遍历，适合快速原型和本地开发。

🌐

P2PBackend

P2P

基于 libp2p 0.53 Kademlia DHT，分布式、去中心化。 内容通过 CID 寻址，节点自动发现和路由。

☁️

CloudS3Backend

Cloud

AWS S3 / MinIO 兼容存储。完整 SigV4 签名实现， 支持自定义端点和路径前缀。init 时验证连接。

🔗

DecentralizedStorage

IPFS

IPFS HTTP API 集成。CID 映射持久化到 JSON。 离线时本地缓存，联网后自动同步到 IPFS 网络。

快速开始

简洁的 API

三行代码开始使用任意存储后端

main.rs 
use usp_core::{StorageHub, StorageOptions, BackendConfig};
use usp_core::backends::{LocalBackend, StorageBackend};
use std::sync::Arc;
use bytes::Bytes;

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    // 创建存储 Hub
    let hub = StorageHub::new();

    // 注册本地后端
    let local = Arc::new(LocalBackend::new("./data".into()));
    local.init(BackendConfig::Default).await?;
    hub.register_backend(local).await;

    // 存储数据
    let receipt = hub.put(
        "my-key",
        Bytes::from("Hello, USP!"),
        StorageOptions::default(),
    ).await?;

    // 读取数据
    if Some(data) = hub.get("my-key").await? {
        println!("Retrieved: {}", String::from_utf8_lossy(&data));
    }

    Ok(())
}

Rust API

StorageHub 是所有存储操作的主入口。 注册后端后，通过统一的 put / get / delete 接口操作数据。

策略路由

根据键名模式、文件大小、标签自动选择最优后端。 小文件走本地，大文件上云，热数据用 P2P 缓存。

命令行工具

CLI 用法

简洁的命令行界面管理存储

# 初始化存储（默认本地后端）
$ ./usp init
# 指定后端类型
$ ./usp init --backend p2p
$ ./usp init --backend s3
$ ./usp init --backend decentralized
$ ./usp store mykey /path/to/file.txt # 存储文件
$ ./usp get mykey /output.txt # 读取文件
$ ./usp delete mykey # 删除文件
$ ./usp list # 列出所有键
$ ./usp stats # 查看存储统计
$ ./usp backend --info # 查看后端信息

灵活配置

支持 .usp.toml 配置文件和环境变量覆盖

环境变量 描述 默认值
USP_DATA_DIR 数据存储目录 .usp-data
USP_S3_ENDPOINT S3 API 端点 —
USP_S3_REGION AWS 区域 us-east-1
USP_S3_BUCKET S3 桶名称 —
USP_S3_ACCESS_KEY 访问密钥 ID —
USP_S3_SECRET_KEY 秘密访问密钥 —
USP_IPFS_GATEWAY_URL IPFS Gateway 地址 https://ipfs.io/ipfs/
USP_IPFS_API_URL IPFS API 地址 http://127.0.0.1:5001
